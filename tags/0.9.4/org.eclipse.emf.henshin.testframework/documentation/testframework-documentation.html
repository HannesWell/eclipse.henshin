Henshin Loaders

(HenshinLoaders.java) Functions for loading henshin files and model graphs

TransformationSystem loadHenshin(String fileName)

Loads a henshin file and returns a TransformationSystem.

EmfEngine loadEngine(String modelFileName, String modelFileExt)

Loads the model from the file and initializes a new EmfEngine on the model.

EmfGraph loadGraph(String modelFileName, ModelFileExt)

Loads the model from the file and returns the corresponding EmfGraph

EmfGraph loadEmbeddedGraph(String embeddedGraphName, TransformationSystem ts)

Loads the specified embedded graph from the TransformationSystem.
Embedded graphs are specified directly in the .henshin file. Using them is not recommended.

EmfGraph loadGraph(String embeddedGraphName, TransformationSystem ts)

See loadEmbeddedGraph

EmfGraph loadGraph(URI uri)

Load an EMF graph from a URI

assertGraphsEqual(EmfGraph graph1, EmfGraph graph2, double matchSimilarityThreshold)

Assert that both graphs are equal. This method uses EmfCompare to compare the graphs. MatchSimilarityThreshold is an internal parameter which can be used to fine-tune the outcome of this comparison. Usually, 0.9 or 0.8 is sufficient. Occasionally, lower values may be needed. Please note that changes in edge order will most likely lead to differences being found, even if henshin should ideally ignore the edge order.

This assertion can be used to quickly assert transformations of graphs are executed correctly, by comparing the current graph (after the transformation has been applied) with a graph containing the expected results loaded from disk.

assertTransformsSubgraph(Rule r, EmfGraph graph, EmfGraph subgraphBefore, EmfGraph subgraphAfter, boolean ignoreChanges, double matchSimilarityThreshold)
assertTransformsSubgraph(Rule r, EmfEngine engine, EmfGraph subgraphBefore, EmfGraph subgraphAfter, boolean ignoreChanges, double matchSimilarityThreshold)
assertTransformsSubgraph(TransformationUnit tu, EmfGraph graph, EmfGraph subgraphBefore, EmfGraph subgraphAfter, boolean ignoreChanges, double matchSimilarityThreshold)
assertTransformsSubgraph(TransformationUnit tu, EmfEngine engine, EmfGraph subgraphBefore, EmfGraph subgraphAfter, boolean ignoreChanges, double matchSimilarityThreshold)
assertTransformsSubgraph(UnitApplication ua, EmfGraph subgraphBefore, EmfGraph subgraphAfter, boolean ignoreChanges, double matchSimilarityThreshold)

This assertion is used if the graph involved in a graph transformation is too big, too complex too random to use assertTransformsGraph. If this is not the case, assertTransformsGraph should be used, as assertTransformsSubgraph is slower and more error-prone.
This assertion will check if subgraphBefore is contained in the graph before the graph transformation (by executing a Rule or TransformationUnit) takes place. Then, the graph is transformed and the assertion will check if subgraphAfter is contained in the graph.
By setting the parameter ignoreChanges, changes to attributes of objects in the graph are ignored, which can be useful if only the graph's structure is of interest.
Please note that this assertion can only check if a subgraph is contained in another graph and is not able to assert that only objects in subgraphBefore were transformed into subgraphAfter by the graph transformation.
This assertion will fail if the Rule/TransformationUnit/UnitApplication can not be applied/executed.

subgraphBefore
hsObj C
hsObj Q
subgraphAfter
hsObj B
hsObj C

Graph before:
hsObj A
hsObj B
hsObj C

OCL Queries

Assertions for results of context-free OCL queries.

assertElementsInQueryResult(String contextFreeOclQuery, EmfGraph graph)

Assert that the context-free OCL query executed on the graph will lead to a result containing elements (i.e. the query result is not empty).

assertObjectInQueryResult(EObject eobj, String contextFreeOclQuery, EmfGraph graph)

Assert that the context-free OCL query executed on the graph will lead to a result containing the specified EObject.

assertObjectNotInQueryResult(EObject eobj, String contextFreeOclQuery, EmfGraph graph)

Assert that the context-free OCL query executed on the graph will lead to a result which does not contain the specified EObject. This will also be the case if the result is empty. If the result should be non-empty, use assertElementsInQueryResult beforehand.

assertGroupInQueryResult(Collection<? extends EObject> group, String contextFreeOclQuery, EmfGraph graph)

Assert that all elements contained in the Collection group are contained in the query result which is obtained by executing the context-free OCL query on the graph.

assertGroupNotInQueryResult(Collection<? extends EObject> group, String contextFreeOclQuery, EmfGraph graph)

Assert that at least one element contained in the group is not contained in the query result.

assertNoElementFromGroupInQueryResult(Collection<? extends EObject> group, String contextFreeOclQuery, EmfGraph graph)

Assert that no element contained in the group is contained in the query result.

assertAtLeastOneElementFromGroupInQueryResult(Collection<? extends EObject> group, String contextFreeOclQuery, EmfGraph graph)

Assert that at least one element contained in the group is contained in the query result.

Parameters

Assertions for Parameters.

assertParameterMappingContainsObject(Match ma, Object obj)

Assert that a parameter mapping of the Match contains the object.

assertParameterMappingEquals(Match ma, String parameterName, Object obj)

Assert that a parameter mapping of the Match contains a parameter with the specified name and the specified object as the value.

TransformationUnits

General assertions for Transformation Units. For assertions on specific objects matched, see Matches.

Please note: All of these assertions will execute the Transformation Unit, so if changes to the graph are not desired, undo() has to be called manually after the assertion.

assertTransformationUnitCanBeExecutedMultipleTimes(TransformationUnit tu, EmfGraph graph)
assertTransformationUnitCanBeExecutedMultipleTimes(TransformationUnit tu, EmfEngine engine)
assertUnitApplicationCanBeExecutedMultipleTimes(UnitApplication ua)

Assert that the specified TransformationUnit/UnitApplication can be executed multiple times, i.e. the first execution resulted in Rules being applied and the second execution resulted in Rules being applied.

assertTransformationUnitCanBeExecutedNTimes(TransformationUnit tu, EmfGraph graph, int n)
assertTransformationUnitCanBeExecutedNTimes(TransformationUnit tu, EmfEngine engine, int n)
assertUnitApplicationCanBeExecutedNTimes(UnitApplication ua, int n)

Assert that the specified TransformationUnit/UnitApplication can be executed at least n times, i.e. n executions each lead to Rules being applied.

assertTuCanBeExecuted(UnitApplication ua)

Execute the specified Unit Application and assert that it could in fact be executed.

Rules

General assertions for Rules. Contains basic statistical assertions for Matches. If more in-depth analysis of Matches (i.e. objects matched) is required, see Matches.

assertRuleHasMatch(Rule r, EmfEngine engine)
assertRuleHasMatch(Rule r, EmfGraph graph)
assertRuleHasMatch(RuleApplication ra)

Assert that the specified Rule produces a match (at least one) on the specified (engine's) graph.

assertRuleHasNoMatch(Rule r, EmfEngine engine)
assertRuleHasNoMatch(Rule r, EmfGraph graph)
assertRuleHasNoMatch(RuleApplication ra)

Assert that the specified Rule produces no match on the specified (engine's) graph.

assertRuleHasNMatches(Rule r, EmfEngine engine, int n)
assertRuleHasNMatches(Rule r, EmfGraph graph, int n)
assertRuleHasNMatches(RuleApplication ra, int n)

Assert that the specified Rule produces exactly n matches.

assertRuleCanBeAppliedMultipleTimes(Rule r, EmfEngine engine)
assertRuleCanBeAppliedMultipleTimes(Rule r, EmfGraph graph)
assertRuleCanBeAppliedMultipleTimes(RuleApplication ra)

Assert that the specified Rule can be applied multiple times (that is, at least twice) on the (engine's) graph. Please note that the Rule will be applied on the graph, so if changes to the graph are not desired, undo() needs to be called manually after this assertion.

assertRuleCanBeAppliedNTimes(Rule r, EmfEngine engine, int n)
assertRuleCanBeAppliedNTimes(Rule r, EmfGraph graph, int n)
assertRuleCanBeAppliedNTimes(RuleApplication ra, int n)

Assert that the specified Rule can be applied at least n times. Please note that the Rule will be applied on the graph, so if changes on the graph are not desired, undo() needs to be called manually after this assertion.

assertRuleExecutedByUnitApplication(UnitApplication ua, Rule r)

Assert that the specified Rule was applied when the UnitApplication was executed. Please note: This assertion requires a UnitApplication that has already been executed and will not execute the UnitApplication.

assertRuleExecutedByUnitApplicationNTimes(UnitApplication ua, Rule r, int n)

Assert that the specified Rule was applied exactly n times when the UnitApplication was executed. Please note: This assertion requires a UnitApplication that has already been executed and will not execute the UnitApplication.

assertRuleCanBeApplied(Rule r, EmfEngine engine)
assertRuleCanBeApplied(Rule r, EmfGraph graph)
assertRuleCanBeApplied(RuleApplication ra)

Assert that the Rule can be applied on the graph. Please note: This will apply the Rule, so if changes on the graph are not desired, undo() needs to be called manually after this assertion. However, this assertion is primarily meant as a replacement for Rule.apply(), as it will apply the Rule and assert that it was applied successfully.
